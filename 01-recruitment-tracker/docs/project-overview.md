# Project Overview — Recruitment Tracker

## Background

As an MBA student specialising in HR and Business Analytics, I developed this tool to practice applying data thinking to a real-world HR process. Recruitment is often managed through emails and spreadsheets without structure — this tracker brings order to that chaos.

## Design Decisions

- **Excel over a database** — Chosen for accessibility; any HR professional can open and use it without technical setup
- **Dropdown menus** — Ensure data consistency and make filtering/pivoting easier downstream
- **Stage-based layout** — Mirrors real recruitment workflows (Sourced → Screened → Interviewed → Offered → Hired/Rejected)

## What I Learned

- How to design data entry tools that stay clean at scale
- The importance of standardising status fields for accurate KPI tracking
- How Excel-structured data can feed directly into Power BI dashboards

## Next Steps / Potential Improvements

- Connect to a Power BI dashboard for real-time visual reporting
- Add a "Reasons for Rejection" field to analyse drop-off patterns
- Introduce a scoring column to quantify candidate fit
